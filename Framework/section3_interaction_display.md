# 第三小节：系统交互展示及说明

## 3.1 交互设计概述

FacadeRemake 采用 **编辑器-运行时双模分离** 交互架构。用户以叙事设计师角色进入 Design 模式，通过可视化蓝图构建故事框架；随后以体验者角色切入 Play 模式，通过 WebSocket 连接后端 LLM 引擎，进行沉浸式交互叙事。两套模式共享同一数据源，交互范式完全独立——前者强调结构化编辑与图形化操作，后者聚焦自然语言对话与实时状态反馈。

***

## 3.2 Design 模式：蓝图叙事编辑器

### 3.2.1 整体布局与核心操作路径

Design 模式采用三区布局：中央 React Flow 蓝图画布、右侧 Inspector 属性面板、工具栏及资源面板。用户典型操作路径为：

1. **双击画布空白区域**创建 Landmark 叙事节点，默认标题为"新节点"，Inspector 自动切换至该节点属性 Tab
2. 在 Inspector 中编辑节点的标题、描述、Phase Tag 阶段标签、Max Storylets 上限、是否结局等字段，所有修改实时同步至画布
3. 从节点右侧 Handle 拖拽至目标节点左侧 Handle，创建过渡连线——系统自动弹出连线属性编辑区，可选择条件线（绿色实线）、计数线（黄色虚线）、兜底线（红色虚线+闪烁）或回合限制线（橙色虚线）
4. 在 Inspector 底部 Storylet Pool 中新建叙事片段，通过弹出模态框编辑触发条件、LLM 指令、效果和重复策略
5. 通过工具栏保存项目快照至 localStorage 或导出 JSON

\[此处插入截图：Design模式全景界面——蓝图画布、节点与连线、Inspector属性面板]

### 3.2.2 蓝图画布交互机制

画布基于 @xyflow/react v12，关键交互包括：

- **节点操作**：以 Windows 3.1 复古风格（渐变标题栏、bevel-out 凹凸边框）呈现，结局节点为面板黄底色。支持单选、Ctrl/Shift 多选、框选和拖拽移动，松手时触发 undo 快照
- **连线管理**：四种连线类型以颜色区分，条件线显示标签文本，右键可删除
- **画布导航**：滚轮缩放（0.3x–2x）、中键拖拽平移（自定义实现）、MiniMap 缩略图
- **快捷键**：Delete/Backspace 删除选中节点，多选时对话框列出全部节点名确认批量删除

## 3.3 Play 模式：沉浸式交互叙事

### 3.3.1 会话初始化

点击工具栏 Play 按钮后，系统执行五步初始化：状态清空→WebSocket 连接至 `ws://localhost:8000/ws/play`（携带唯一 session\_id）→等待后端 `ready` 信号→发送 `init_scene` 场景数据（Landmarks、Storylets、Characters、WSD）→后端启动 GameEngine，前端显示 TypingIndicator 跳动圆点动画。连接状态通过 ConnectionBanner 实时显示（绿/琥珀/红色信号灯），5 秒超时则展示连接失败错误界面。

\[此处插入截图：Play模式初始化界面——连接横幅、系统时钟、等待指示器]

### 3.3.2 对话交互

核心交互区由 SceneStage 立绘舞台和 NarrativeBox 对话流构成。立绘舞台以网格背景呈现 Trip 和 Grace 双角色，当前说话者头部显示黄色闪烁 speakingPulse 标记。对话流采用交互式小说风格，五种角色以专属颜色呈现——Trip 深红、Grace 海军蓝、玩家森林绿、旁白深棕、系统消息紫色。每条消息含三项维度：speech（台词）、action（动作，斜体+星号包裹）、thought（内心独白，仅调试面板查看）。最新消息具备**逐字打字机效果**（20ms/字），历史消息即时完整显示，消息列表以 requestAnimationFrame 驱动平滑滚底。

\[此处插入截图：Play模式对话界面——角色发言逐字渐入效果、输入栏"等待回应中"状态]

### 3.3.3 玩家输入系统

底部 CommandBar 提供完整输入功能：单行文本输入框（左侧黑色 "INPUT>" 终端风格标签），Enter 发送文本或空输入表示沉默（显示省略号斜体）；发送按钮 loading 时变灰且不可点击；回退按钮显示可回退步数（最多 30 步），一键恢复至上一玩家输入前的完整状态；重置按钮经确认对话框后重建会话。

\[此处插入截图：CommandBar输入区细节——发送、回退、重置按钮组]

### 3.3.4 运行时调试与状态编辑

右侧 Debug Panel 可折叠，提供四项能力：① 实时查看当前 Landmark/Storylet 叙事状态；② 通过 NumberBar 滑块和 FlagToggle 开关手动编辑 WorldState（颜色随值渐变：绿→黄→红）；③ 聚合最近 3 条内心独白卡片（角色色条标识）；④ LLM 调用日志以请求/响应对展开。所有修改通过 `debug_worldstate` 消息实时同步后端，保证 LLM 上下文一致。

\[此处插入截图：Debug面板展开——LLM日志折叠区、品质滑块、FlagToggle开关]

## 3.4 位置系统导航

左侧 LocationPanel 以 BFS 算法计算并渲染可达位置列表：当前位置为绿色高亮+呼吸光效+标签，相邻位置提供"前往→"标签（hover 右滑动效），不可达位置降低透明度并提示"距离较远"。每个位置下方以双色徽章展示角色分布和物品分布。点击相邻位置即时通过 `move_location` 消息同步后端，触发 `location_update` 更新。
