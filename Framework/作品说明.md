## 一、作品概述

FacadeRemake 是一个以互动叙事为核心的体验型数字作品，以经典互动叙事游戏 Facade（Mateas & Stern，2005）为灵感，在当代大语言模型与多智能体协作技术的支撑下进行了系统性重建与创新拓展。
作品的核心命题来源于互动叙事领域长期悬而未决的根本矛盾：叙事需要连贯，但玩家需要自由。传统分支叙事依靠预设路径保证可控性，却面临内容数量随分支深度指数级爆炸的不可扩展困境；纯粹的涌现式叙事赋予玩家充分的行动自由，却极易导致情节偏离、戏剧张力消散。FacadeRemake 探索了第三条路径——将预设计叙事骨架与运行时多智能体内容生成融合为一套混合架构，力图在结构约束与即兴涌现之间找到工程上可行、体验上令人满意的平衡点。
作为毕业设计的互动体验与效果呈现，本作品提供了完整可运行的系统原型。体验者可在浏览器中通过 Design 模式对故事世界进行创作编辑，再即时切换至 Play 模式进入沉浸式自然语言对话，验证当前叙事设计的实际效果。

## 二、核心设计理念

### 2.1 设计出发点

互动叙事的本质冲突可以用一个简单的公式衡量：若每个决策点提供 $n$ 个选项、叙事深度为 $d$ 层，则路径总数为 $n^d$。当 $n=3$、$d=10$ 时，故事路径已超过五万条，远远超出人工内容创作的可行范围。这意味着传统手工编写分支的方式在规模上存在天花板。

与此同时，完全开放的 LLM 生成叙事（如 AI Dungeon 的实践）在研究层面已经证明：过高的自由度会导致玩家缺乏目标感，叙事随时间逐渐失去核心戏剧张力，重复游玩率显著下降。

FacadeRemake 的设计起点正是这个张力场：**如何让 LLM 充分发挥内容生成能力，同时保证故事不脱离作者预设的叙事弧线？**

### 2.2 核心原则："骨架管发生什么，Agent 管怎么发生"

系统架构遵循一条统一的设计原则——叙事骨架层负责定义故事"**发生什么**"，包括阶段推进顺序、关键转折与结局走向；多智能体内容生成层负责实现"**如何发生**"，通过 LLM 驱动的角色对话、情感表达和情境细节，在骨架约束的边界内产生丰富的涌现内容。

这一分离设计借鉴了 Triangle Framework 理论（Senanayake, 2025），将叙事设计空间明确划分为三个维度：
- **作者控制（Authorial Control）**：通过叙事骨架层的 Landmark 有向无环图实现；
- **系统涌现（System Emergence）**：通过多智能体协作的动态内容生成实现；
- **玩家代理（Player Agency）**：通过自然语言自由输入机制实现。

### 2.3 戏剧原型：向 Facade 致敬

Facade（2005）是互动叙事史上的里程碑——它首次实现了玩家以完全自然语言与 AI 角色进行深度情感对话。作品中，玩家扮演造访老友的第三者，被卷入 Trip 与 Grace 这对夫妻正在崩溃的婚姻危机。Facade 证明了"自然语言交互的互动戏剧"在技术上是可能的，但其实现依赖海量人工编写的联合对话行为（Joint Dialog Behaviors），几乎无法移植到新的叙事场景。
FacadeRemake 以同一故事场景为载体，探索用当代 LLM + 多智能体框架重建并扩展这种体验的可行路径。在保留原作"进门即是危机现场"的戏剧张力的基础上，本作品实现了**可泛化的技术框架**——叙事设计师无需编写任何代码，仅在可视化编辑器中配置叙事结构与角色数据，即可生成截然不同的互动叙事体验。

## 三、系统架构与核心模块

### 3.1 双层叙事引擎：Landmark + Storylet

系统以 **Landmark-Storylet 双层叙事引擎**作为核心架构基础。

**Landmark 层**（宏观叙事骨架）以有向无环图（DAG）组织故事的核心阶段节点，定义每个阶段的进入条件、转场触发机制与禁止揭露内容。本作品内置的 Dinner Party 场景包含七个 Landmark 节点，构成一个具有分支结构的叙事 DAG：

```
lm_1_arrive（做客初见）
    ↓
lm_2_cracks（关系裂缝）
    ↓          ↓
lm_3a_trip  lm_3b_grace
（Trip坦白）（Grace揭露）
    ↓          ↓
lm_4_resolve（摊牌与抉择）
    ↓              ↓
lm_5a_reconciliation  lm_5b_breakup
（和解结局）         （破裂结局）
```

DAG 结构保证叙事始终向前推进，不会陷入循环；分支节点实现了根据玩家行为走上不同叙事路径的能力，同时所有路径在关键节点汇聚，保证戏剧弧线的完整性。

**Storylet 层**（场景级叙事单元）在 Landmark 约束的范围内动态运作。每个 Storylet 拥有明确的叙事目标、前置条件、显著性（Salience）评分机制和后置效果。Storylet 的选择通过三层过滤流程实现：标签过滤（确保阶段一致性）→ 条件过滤（确保前置依赖满足）→ Salience 评分（根据世界状态选择最契合当前情境的场景）。本作品实现了包含十五个 Storylet 叙事单元的完整场景池，覆盖从初次相见到四个差异化结局的全部叙事弧线。

### 3.2 多智能体内容生成框架

内容生成层由三类职责明确的 Agent 构成，以"戏剧隐喻"划分角色：

**InputParser（舞台监督）** 是玩家输入进入系统的统一入口。采用双层验证流水线：规则层以零 LLM 调用成本快速过滤元对话（"你是 AI 吗"）、物理违规等异常情况；语义层将场景约束传入 LLM 进行情境敏感的合法性判断，同时完成语义条件匹配——两项任务合并为单次 LLM 调用，兼顾准确性与性能。

**DirectorAgent（叙事导演）** 参考 IBSEN 框架的导演-演员分离架构，但在控制粒度上进行了关键调整：本系统的 DirectorAgent 仅输出**指导性指令**（情绪基调、叙事节拍方向、禁止话题），而非具体台词，为角色 Agent 保留充分的生成自由度。内部的 GoalTracker 组件实时追踪叙事目标进度，在推进滞后时自动触发 accelerate（加速推进）模式干预；InstructionGenerator 生成包含 primary_goal、tone_guidance、narrative_beat、forbidden_topics 等七个维度的结构化导演指令。

**CharacterAgent（角色演员）** 为 Trip 与 Grace 各配置独立实例，每个实例维护完整的角色配置文件，包括身份设定、性格特征、背景故事、秘密知识（仅对该角色可见）、独白模板与专属行为库。角色响应通过**三步生成架构**产出：
1. **内心独白生成**：从 IBSEN 独白模板库选取匹配模板，生成内心真实想法（不直接说出口，但驱动后续表达）；
2. **行为选择**：从预定义的行为库（如 Trip 的 `admit`、`get_angry`、`deflect`，Grace 的 `cold_truth`、`controlled_sarcasm`、`withdraw`）中选取最符合当前情境的行为模式；
3. **台词与动作生成**：在内心独白、行为模式与导演指令的多重约束下，通过 LLM 生成具体对话内容。

三步架构的核心创新在于实现了戏剧所要求的"**心口不一**"效果——角色在表面台词与内心真实想法之间形成张力，是叙事深度的重要来源。

### 3.3 世界状态（WorldState）叙事黑板

三类 Agent 之间不直接通信，而是通过共享 **WorldState 叙事黑板**实现间接协作。WorldState 管理三类状态变量：

- **Qualities**（数值型）：如 `tension`（婚姻紧张度）、`grace_trust`、`trip_trust`（对玩家的信任度），驱动叙事节奏与转场判断；
- **Flags**（布尔型，单向写入）：如 `trip_confessed`、`secrets_revealed`，记录不可逆的叙事里程碑事件；
- **Relationships**（关系型数值）：记录角色对玩家的态度变化，正值表示好感，负值表示疏远。

WorldState 采用观察者模式与事务机制，支持状态更新的原子性操作与变更通知，实现前端实时同步。

### 3.4 异步事件驱动引擎

叙事推进模块基于 Python asyncio 三轨并发架构实现：
- **叙事轨道（Narrative Push Loop）**：按 BeatPlan 序列自动推进剧情；
- **输入轨道（Player Input Loop）**：监听玩家自然语言输入；
- **事件消费轨道（Event Consumer）**：分发事件至 GameEngine 执行处理。

三轨之间通过 `asyncio.Event` 信号机制同步，LLM 调用通过 `run_in_executor` 放入线程池，避免阻塞主事件循环，保证游戏交互的流畅响应。

## 四、体验设计：Design 模式与 Play 模式

FacadeRemake 提供两种互补的交互模式，体现了"**创作者视角**"与"**玩家视角**"的完整体验闭环：

### 4.1 Design 模式——故事世界创作编辑

Design 模式是面向叙事设计师的可视化创作工具。

**Landmark 节点图编辑器**基于 React Flow 实现，支持：
- 节点的拖拽定位与多选操作；
- 通过连线建立叙事阶段之间的转场关系；
- 以颜色区分叙事阶段类型（act1 蓝色、act2 橙色、结局节点绿色）；
- TransitionEdge 以线条样式区分转场类型（条件转场实线、兜底转场虚线）。

**Inspector 属性编辑面板**提供四个核心子面板：
- **TransitionsTab**：编辑转场条件（世界状态条件、语义条件、回合限制）；
- **StoryletPool**：管理场景叙事单元，配置前置条件、叙事目标、Salience 评分与后置效果；
- **CharactersPanel**：配置角色档案，包括角色身份、性格描述、背景故事、秘密知识、IBSEN 独白模板与行为库；
- **WorldStatePanel**：定义 qualities、flags 与 relationships 的初始值与取值范围。

**支持最多 50 步的撤销/重做操作**，所有编辑操作实时保存至前端状态管理（Zustand），切换至 Play 模式时通过 init_scene 消息将完整场景配置下发至后端，实现 Design → Play 的**即时验证**。

Play 模式核心交互设计特性：

**自然语言自由输入**：玩家以任意自然语言表达意图，无需选择预设选项。输入通过 InputParser 的双层验证后，语义匹配结果同时驱动 Storylet 选择与 Landmark 推进判断。

**角色表现力**：CharacterAgent 生成的内心独白以可折叠的"💭"标签形式附在角色消息下方。玩家可点击展开查看角色内心真实想法与表面台词之间的反差，感受戏剧叙事独有的心理层次张力。

**实时世界状态反馈**：左侧面板的状态数值（如 TRIP张力、场景紧张度）随每轮对话实时更新，玩家的行为对叙事走向的影响以可量化的方式呈现，强化了"玩家行为有意义"的代理感。

**沉默机制**：玩家直接按回车（空输入）将被识别为沉默行为，触发角色的催促反应或自主推进，使沉默本身成为有叙事意义的互动行为。

**Debug 面板**（右下角，可展开）：实时展示每次 LLM 调用的请求参数与响应详情，LLM 响应延迟以毫秒级时间戳标注（如截图中 T2 LLM Trip 回应生成 · 1.2s），支持开发者对生成过程进行全链路透明监控。

**30 步回退机制**：玩家可随时撤销上一轮输入，回退至上一轮的对话状态重新尝试，降低玩家因错误输入导致叙事走向偏离的挫败感。

## 五、核心叙事内容设计

### 5.1 场景蓝本：Dinner Party

FacadeRemake 内置的叙事场景以 Facade 原作的"餐厅聚会"情境为蓝本，并进行了本土化改编：

**叙事设定**：玩家扮演 Trip 与 Grace 的老友，在一个普通的周末晚上应邀造访。客厅里弥漫着一种不自然的安静——Trip 站在窗边，背对着门口；Grace 在厨房走来走去，避免与 Trip 目光相对。玩家意识到，自己撞见了这段婚姻正在崩塌的边缘。

**四个核心叙事节点**：
1. **做客初见（lm_1_arrive）**：表面的寒暄与微妙的紧张，故事的引子；
2. **关系裂缝（lm_2_cracks）**：争端开始浮出水面，玩家感受到隐藏的张力；
3. **坦白/揭露（lm_3a/3b）**：根据玩家行为，或 Trip 主动坦白，或 Grace 揭露秘密——两条路径各自展现不同的戏剧逻辑；
4. **摊牌与抉择（lm_4_resolve）**：所有秘密摊上桌面，玩家的调解或沉默将影响最终走向。

**四个差异化结局**：
- **和解**（玩家积极调解 + 双方信任度达标）；
- **破裂**（矛盾激化，关系无法修复）；
- **悬而未决**（玩家沉默旁观，两人陷入冰点）；
- **意外揭示**（玩家触发特定语义条件，引出意料之外的第三方秘密）。

### 5.2 角色设计

**Trip** 是一个外表强势、内心焦虑的丈夫。他的行为库涵盖从 `get_angry`（情绪爆发）到 `admit`（承认真相）的全谱，关键秘密知识被锁定在其角色提示词中，仅在满足特定叙事条件时才会在对话中浮现。

**Grace** 是一个以克制掩盖痛苦的妻子。其专属行为 `cold_truth`（冷静揭示）与 `controlled_sarcasm`（克制讽刺）赋予她独特的对话质感，`withdraw`（情感撤退）与 `break_down`（情绪崩溃）之间的张力构成了她最有戏剧性的叙事弧线。

两个角色的 IBSEN 独白模板设计，使得他们在说话时总是"说一半留一半"，玩家需要通过解读言外之意来理解真实处境——这是本作品在叙事体验层面最核心的设计追求。

## 八、作品意义与创新点

### 8.1 理论意义

本作品在实践层面验证了以下理论假说：

1. **Storylet-QBN 模型与多智能体 LLM 框架可以有效整合**：传统 Storylet 系统依赖手工编写固定文本，本作品将 Storylet 的结构控制能力与 LLM 的动态内容生成能力结合，在保留叙事可控性的前提下实现了真正的内容涌现；

2. **"间接控制"优于"直接脚本"**：DirectorAgent 仅输出方向性指令而非具体台词的设计，在保证叙事方向的同时为角色保留了充分的生成自由度，验证了"指导性控制"在叙事质量与角色自然度之间的优越性；

3. **心口不一架构对叙事深度有实质性贡献**：三步生成中的内心独白层，在不增加玩家认知负担的前提下，为角色对话增添了显著的人物厚度与戏剧张力。

### 8.2 实践意义

本作品提供了一个**可复用的互动叙事技术框架**：任何具备叙事设计能力的创作者，都可以通过 Design 模式配置新的场景数据，驱动同一套后端引擎运行完全不同的互动叙事体验，而无需具备工程开发能力。这在互动叙事创作工具的易用性与能力边界上，相较于现有工具（如 Twine、Ink）向前迈出了重要一步。
