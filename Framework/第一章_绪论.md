# **1**       **章节一** **绪论**

## 1.1    研究背景及意义

互动叙事（Interactive Narrative）是一种数字互动体验的形态，其核心目标是将用户沉浸于一个虚拟世界，使用户相信自己正置身于一个正在展开的故事当中，且其行为能够对该戏剧性故事产生影响，改变其剧情发展方向甚至是最终结局。然而，这一目标面临着一个根本性的矛盾：叙事连贯性来自于预先设定的事件序列与因果关系，而用户能动性（User Agency）则要求赋予用户行为以真实的影响力。如何在保障叙事体验连贯性的前提下，为用户的自由行为提供有意义且一致的响应，是互动叙事所面对的核心挑战[1]。

早期的互动叙事系统大多依赖于有限状态机驱动的分支对话术[1]，随后研究者引入自动规划方法，试图用规划器实时生成满足叙事约束的行动序列[2]。2005年，Mateas与Stern开发的Façade系统将自然语言理解与实时情感表达融入互动叙事场景，使玩家能够以自然语言对话的方式与两个AI角色进行深度交互[3]。然而，这些技术路径本质上仍属人工预设——过于依赖设计师手工编写分支情节，在规模较大的叙事场景下难以兼顾灵活性与连贯性。

近年来，以GPT系列为代表的大语言模型（Large Language Model, LLM）在自然语言理解、上下文推理和多轮对话方面取得了显著进展[4,5]，为互动叙事系统的内容生成提供了新的可能。与传统的基于规则的系统相比，LLM可以根据当前情境的上下文实时生成连贯且富有表现力的对话，无需为每一句台词预设固定文本。然而，完全基于LLM的开放式叙事缺乏全局的叙事把控能力，容易导致情节发展陷入混乱[6,7]。而多智能体协作则通过将叙事任务分解为多个专业化的子智能体任务协作执行，实验证明，系统可以在维持叙事一致性与角色自主性之间取得更好的平衡[8–11]。

基于上述背景，本研究提出并实现了一个基于多智能体协作的自适应游戏剧情分支系统，以经典的互动叙事游戏Façade的核心场景为载体，探索LLM与叙事控制理论相结合的技术路径。系统的设计思路是：在叙事骨架层（Landmark）保证作者叙事意图不受损害的前提下，通过多智能体协作系统基于Storylet叙事单元实时动态生成符合角色设定与故事情境的对话内容，使得玩家能够在自然语言交互中获得真实而有意义的叙事体验。

本研究的理论意义在于：将传统的Storylet-QBN（Quality-Based Narrative）叙事模型与多智能体协作框架进行整合，探索LLM驱动的互动叙事系统在叙事控制与内容涌现之间的平衡机制。实践意义在于：为下一代互动叙事游戏的开发提供一个可复用的技术框架与实现参考。

## 1.2 国内外研究现状

### 1.2.1 传统互动叙事系统

互动叙事系统的研究可追溯至20世纪90年代初期，主要聚焦于如何在叙事连贯性与玩家自由度之间取得平衡。根据Riedl[1]等人提出的三维度分类框架，互动叙事系统可以从以下三个维度进行分析：作者意图（Authorial Intent）——作者对叙事结构的预设程度，以及系统可偏离作者设计意图的幅度；角色自主性（Character Autonomy）——NPC是否由系统全局控制，或能以自治的方式行动；玩家建模（Player Modeling）——系统是否根据玩家行为建立偏好模型并据此调整叙事体验。其中，作者意图维度直接决定系统的叙事生成方式，是传统互动叙事系统最核心的区分标准。基于该维度，可将经典互动叙事系统分为三类：强作者意图系统、混合系统与生成式系统。

第一类：强作者意图系统（Manually Authored）。该类系统中，所有可能的叙事路径均由作者预先设计，玩家只能在有限的选项中进行选择。故事结构通常为分支故事图（Branching Story Graph），每个节点代表一段固定的叙事内容，玩家通过选择来决定后续的路径。这一模式的代表形式包括CYOA（Choose-Your-Own-Adventure）书籍与超媒体叙事（Hypermedia Narratives）。其优点在于故事质量高，完全符合作者的创作意图、实现简单；然而随着交互点增多，分支的数量呈指数增长，存在严重的组合爆炸问题，使系统难以扩展。同时玩家只能选择预设的选项，自由度较低。

第二类：混合系统（Hybrid Systems）。此类系统在作者意图维度上处于中间位置：作者不再预先编写所有可能的叙事路径，而是提供部分结构化的叙事信息（如关键事件及其约束关系），系统则在此基础上动态调整故事发展。此类系统通常采用情节图（Plot Graph）来表示叙事空间，其中节点（Nodes）对应叙事事件，边（Arcs）表示事件之间的先后或依赖关系。运行时，系统会通过一个搜索过程（Searching Process）在图中选择合适的叙事轨迹，实现方法包括对抗式搜索、强化学习以及基于案例的规划等。混合系统一般包含一个体验管理器（Experience Manager），负责监控玩家行为与世界状态，并在必要时对叙事进行调整，以维持整体连贯性和体验质量。当玩家行为偏离预期的叙事曲线时，体验管理器可以通过调整NPC行为或触发新的事件来重新组织后续情节。与强作者意图系统相比，混合系统在叙事质量与灵活性之间取得了更好的平衡，但其动态调度机制仍然依赖大量的叙事建模与系统设计工作。Mateas[3]等人提出的Facade是此类系统的典型代表。Façade的实现依赖于在 ABL 语言中编写的大量联合对话行为（Joint Dialog Behaviors），而这些行为被组织为一系列的故事节拍（story beats），由戏剧管理器（Drama Manager）进行实时调度。该系统的核心在于其能够将戏剧叙事分解为层次化的故事与行为片段，并在玩家的影响下实时重构戏剧表演。

第三类：生成式系统（Automatically Generated）。该类系统的核心是将一个完整的故事生成器嵌入体验管理器中，使系统能够在运行时从零开始生成叙事。作者提供的知识不再以预设的情节结构呈现，而是以更一般化的形式编码为领域理论（Domain Theory），声明虚构世界中那些动作是可能的、以及在何种世界状态下可以执行，而不包含动作顺序方面额外的约束。叙事的具体展开顺序由生成器根据当前世界状态与玩家行为在运行时动态决定。生成式系统的关键是解决边界问题（Boundary Problem）：当玩家执行偏离系统预期的叙事行为时，体验管理器需要识别该偏离并选择应对策略。可选策略分为两类：干预（Intervention）是指阻止玩家越界，例如通过改变NPC行为或修改虚构世界状态来避免叙事崩溃；适应（Accommodation）则是允许偏离发生，并调用故事生成器重新构建后续叙事。Automated Story Director[12]将故事生成器与半自治角色相结合，使系统能够在保持叙事目的的同时响应玩家行为；C-DRAGer[13]则利用基于实例的规划方法在运行时选择并改写故事实例，以生成新的叙事片段。与强作者意图系统和混合系统相比，生成式系统具有最高程度的叙事灵活性与玩家自由度，但叙事质量高度依赖故事生成器的能力，且系统设计与世界建模的复杂度更高。

### 1.2.2 基于大语言模型的互动叙事系统研究进展

随着大语言模型的兴起，关于互动叙事系统的研究迎来了根本性的范式转变。大规模预训练赋予了LLM丰富的世界知识、指令遵循能力与语义事件追踪能力，使得系统不再依赖预写文本，而是能根据实时的情境动态生成叙事内容。

徐丽芳和左涛[7]以AI Dungeon为案例，分析了其技术基础与叙事特征。AI Dungeon基于OpenAI发布的GPT语言预测模型构建，采用完全开放的叙事结构：故事无固定开端，角色与背景均由玩家自定义，叙事支点处不存在选择限制，也没有预设的故事走向。徐丽芳等人指出该系统的局限：由于模型无法保持长期逻辑连贯，AI Dungeon至多只能根据玩家最近八次输入的文本给出反馈，故事进程较长时难以维持完整且连续的世界观；此外，单一的文字叙事符号缺少音乐、音效和动态影像等富媒体元素，玩家代入感有限。

Yong和Mitchell[6]则从用户研究角度考察了LLM驱动的互动叙事系统的体验特征。他们对十名参与者进行了质性观察研究，让其反复游玩AI Dungeon并记录其游玩行为与认知变化。研究发现，玩家在初次面对LLM驱动的叙事系统时，倾向于假设该系统具有线性或分支叙事结构，并据此在脑海中构建故事世界的心理模型，据此做出行动决策；但随着重复游玩，玩家逐渐意识到系统能够提供的看似无限多的叙事可能性并非真正的叙事深度，过高的自由度使玩家缺乏挑战感，失去玩下去的目标与动力。研究总结了玩家重复游玩的三个动因：叙事闭合（Narrative Closure）——玩家寻求特定叙事线的收束；叙事不一致（Narrative Incoherence）——系统生成的内容出现逻辑断裂；以及每次游玩体验间差异的预期（Expected Variations）——玩家期待在重复游玩中获得不同内容。研究者同时观察到，当叙事闭合或遭遇叙事不一致时，部分玩家不再愿意继续推进剧情，转而开始试探甚至"把玩"系统本身，以非结构化的方式展开行为。

在叙事内容生成方面，Kumaran等人开发的NarrativeGenie系统[14]接受设计师以二十至三十词自然语言描述的叙事弧（Narrative Arc），通过LLM将其转化为包含五至十个关键事件的部分有序事件图，从而在保持作者意图的同时允许玩家以不同顺序探索叙事事件。该系统还引入了自适应对话管理器（Adaptive Dialogue Manager），在游戏运行时根据玩家行为历史动态生成NPC的对话、手势与表情。Kumaran等人还提出SCENECRAFT框架[15]，将设计师的文本指令自动转换为Unity虚拟环境中的可玩场景，同时生成与之匹配的NPC对话内容。Gao等人[16]的研究则表明，LLM生成的对话在逼真度上已接近真人水平。

在降低互动叙事创作门槛方面，Sun等人[17]提出的DRAMA LLAMA框架与传统Storylet系统需要作者构想所有叙事可能性不同，该框架允许作者仅以自然语言定义Storylet的触发条件，由LLM判断条件是否满足并驱动叙事事件激活，从而将作者从复杂的逻辑条件编写中解放出来。用户研究表明，自然语言条件虽然提升了创作效率，但也带来了触发不稳定的问题。在此基础上，同一团队提出的Dramamancer系统[18]进一步将自然语言拓展至Storylet的文本内容与状态效果，使作者全程无需编写任何变量标签。此外，该系统引入了时间触发机制，当玩家在规定回合数内未触发任何推动剧情的叙事事件时，系统将自动激活事件以驱动剧情演进，从而避免叙事停滞。

在 LLM 驱动的叙事系统中，角色之间的协调以及叙事质量与玩家自由度的平衡是核心挑战。多智能体协作为此提供了一条解决路径，其核心思想是将叙事生成任务分解为多个专业化的子任务，由不同智能体分工完成，以避免单一LLM注意力分散与上下文瓶颈的问题。

Han等人[8]提出的IBSEN框架引入了导演-演员（Director-Actor）双层架构来解决自由角色交互导致的叙事漂移问题。该框架中，导演智能体负责维护预定义的剧情目标序列，每轮对话后判断当前目标是否完成，并在必要时向演员智能体发出包含关键词的高级指令，而非直接给出台词；演员智能体则依据角色档案、历史记忆与导演指令自主生成符合角色性格的对话内容。实验表明，移除导演指令后故事逻辑性、连贯性与角色一致性均出现显著下降。

Huot等人[9]则从叙事写作的角度出发，提出了Agents' Room框架。该框架将故事生成流程分为规划层与生成层：规划层包含情节、角色、场景、冲突等多个专业化智能体，分别负责构建故事的骨架结构；生成层则包含起始、上升、高潮、下降、结局等叙事弧各阶段对应的智能体，按顺序将骨架填充为完整文本。两层之间通过共享记事本（Scratchpad）传递中间结果，调度器负责决定调用顺序与时机。人类评估表明，该框架在情节、创意、角色发展与语言质量四个维度上均优于端到端的单一LLM生成方法。

Yu等人[10]提出的多智能体角色模拟框架则关注角色记忆与叙事时间的问题。他们将叙事过程解耦为两层：演绎层（Fabula）以时间顺序让角色智能体按真实发生顺序生成场景内容，从而建立角色记忆；重写层（Syuzhet）则参照演绎层的角色对话历史，按照预设的呈现顺序重新组织叙事文本。这一设计使角色能够在后续场景中引用早期经历，增强了叙事的一致性与深度。

Wu等人[11]则从用户体验的角度切入，围绕沉浸感与能动性两个维度考察LLM驱动的互动戏剧。他们提出了剧作引导生成方法，将经典剧作技巧（如悬念、转折、反讽等）作为启发式原则注入LLM的故事生成过程；同时设计了基于情节的反思机制，使角色智能体能够根据玩家行为动态调整情节走向，增强玩家行为的意义感。实验结果表明，该方法在沉浸感与能动性两个维度上均显著优于基线方法。

综合上述研究可以发现，当前LLM驱动的互动叙事系统主要围绕叙事控制粒度、内容生成质量与玩家能动性表达三个维度展开。本文借鉴Storylet框架的叙事控制机制与多智能体协作的内容生成方法，探索一种兼顾叙事控制能力与内容生成质量的自适应互动叙事系统。
