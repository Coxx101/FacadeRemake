# 叙事参考与故事线设计 v0.1

> FacadeRemake 故事设定参考文档：父母婚姻危机 + 子女视角的经典叙事分析

---

## 一、核心参考作品

### 1. 电影《克莱默夫妇》(Kramer vs. Kramer, 1979)

**导演**：Robert Benton  
**核心设定**：妻子离开后，丈夫独自抚养儿子，后期妻子回来争夺抚养权

**叙事结构分析**：
```
第一幕：失衡
  - 妻子 Joanna 突然离开，Ted 被迫独自照顾儿子 Billy
  - 父子从陌生到建立深厚感情

第二幕：冲突升级
  - Joanna 回来，想要回 Billy
  - 法庭上的抚养权争夺
  - Ted 意识到自己过去作为丈夫的失败

第三幕：和解与放手
  - Ted 为了儿子的幸福，选择放手
  - Joanna 最终也意识到 Billy 更需要父亲
```

**对 FacadeRemake 的启示**：
- 孩子的视角不是"旁观者"，而是**被争夺的对象**
- 父母各自有"合理"的立场，没有真正的反派
- 情感高潮往往在"选择站队"的时刻

---

### 2. 电影《婚姻故事》(Marriage Story, 2019)

**导演**：Noah Baumbach  
**核心设定**：戏剧导演 Charlie 和演员 Nicole 的离婚过程

**叙事结构分析**：
```
开场：双视角独白
  - 先分别讲述"我为什么爱对方"
  - 再讲述"为什么无法继续"
  - 建立"他们曾经相爱"的共情基础

中段：律师介入，冲突升级
  - 原本和平的分离变成战争
  - 孩子 Henry 成为筹码
  - 最亲密的人知道如何伤害对方最深

高潮：走廊对峙
  - Charlie 和 Nicole 私下对话
  - 从争吵到拥抱，展现复杂情感
  - 离婚不代表不爱，只是无法共同生活
```

**对 FacadeRemake 的启示**：
- 开场建立"他们曾经是好父母"很重要
- 孩子（Henry）在场/不在场决定对话走向
- 最痛的冲突往往发生在"私下"，没有孩子在场时

---

### 3. 电影《鱿鱼和鲸》(The Squid and the Whale, 2005)

**导演**：Noah Baumbach  
**核心设定**：从两个青少年的视角看父母离婚

**叙事结构分析**：
```
第一阶段：父母宣布分居
  - 兄弟俩的反应截然不同
  - Walt（16岁）站在父亲一边，鄙视母亲
  - Frank（12岁）更困惑，更情绪化

第二阶段：站队与忠诚分裂
  - 父亲 Bernard（自负的作家）不断贬低母亲
  - Walt 开始模仿父亲的傲慢，伤害母亲
  - Frank 用行为问题表达焦虑

第三阶段：幻灭与和解
  - Walt 意识到父亲并不完美
  - 兄弟俩重新理解母亲
  - 在博物馆看到"鱿鱼与鲸"的展品，象征父母战争的创伤
```

**对 FacadeRemake 的启示**：
- **这是最重要的参考**：子女视角的婚姻危机叙事
- 孩子会不自觉地"站队"，但站队是有代价的
- 父母的"战争"会在孩子身上重演
- 结局不是"问题解决"，而是"接受现实"

---

### 4. 戏剧《玩偶之家》(A Doll's House, 1879)

**作者**：Henrik Ibsen  
**核心设定**：妻子 Nora 意识到自己在婚姻中被当作"玩偶"

**叙事结构分析**：
```
三幕结构：
  第一幕：表面的完美家庭
    - Nora 看似幸福，但隐藏着一个秘密（伪造签名借款）
    
  第二幕：秘密被威胁揭露
    - 丈夫 Torvald 发现真相后的反应暴露了他的自私
    - Nora 意识到他爱的不是她，而是"完美的妻子形象"
    
  第三幕：觉醒与离开
    - Nora 决定离开丈夫和孩子
    - 著名的"关门声"：她选择了自我，而非家庭角色
```

**对 FacadeRemake 的启示**：
- 秘密的揭露是戏剧的核心引擎
- 孩子（虽然戏份少）是母亲选择留下的理由，也是她最终离开的原因
- "觉醒"可以是任何一方：父亲、母亲，或孩子

---

### 5. 戏剧《长夜漫漫路迢迢》(Long Day's Journey Into Night, 1941)

**作者**：Eugene O'Neill  
**核心设定**：奥尼尔自传式家庭悲剧，母亲吸毒、父亲吝啬、兄弟酗酒

**叙事结构分析**：
```
一天之内的四幕：
  第一幕：早晨，表面的平静
  第二幕：下午，母亲的毒瘾被提起
  第三幕：傍晚，兄弟间的冲突
  第四幕：深夜，所有人卸下伪装，真相与痛苦爆发
```

**对 FacadeRemake 的启示**：
- **时间压缩**增强戏剧张力（一天 vs 你的"一顿晚餐"）
- 每个人都有"版本"的家庭故事，互相矛盾
- 孩子（Edmund）既是受害者，也是叙述者
- 没有解决方案，只有"被看见"

## 二、理论框架：Karpman 戏剧三角

Stephen Karpman 在 1968 年提出的**戏剧三角**（Drama Triangle）是理解家庭冲突的核心工具。

### 三个角色

```
        迫害者 (Persecutor)
              △
             / \
            /   \
           /     \
          /   ?   \
         /         \
拯救者 (Rescuer) ———— 受害者 (Victim)
```

| 角色 | 核心信念 | 行为特征 |
|------|---------|---------|
| **受害者** | "我好可怜，都是他们的错" | 无助、抱怨、寻求关注 |
| **迫害者** | "都是你的错，你要负责" | 指责、控制、贬低 |
| **拯救者** | "我来帮你，没有我你不行" | 过度帮助、隐藏控制欲 |

### 在家庭危机中的动态

**经典场景**：
```
母亲（受害者）："我为这个家付出了一切，却没人理解我"
    ↓
父亲（迫害者）："你总是这样，从不反思自己的问题"
    ↓
孩子（拯救者）："别吵了，都是我的错，我以后会乖的"
    ↓
（角色转换）
父亲（受害者）："我工作这么辛苦，回家还要被指责"
母亲（迫害者）："你从来不关心这个家"
孩子（更努力的拯救者）："我考了第一名，你们别吵了好不好"
```

### 对 FacadeRemake 的设计启示

1. **玩家可以扮演任何角色**
   - 选择当"拯救者"：试图调解，但可能适得其反
   - 选择当"受害者"："你们从来不考虑我的感受"
   - 选择当"迫害者"："你们根本不配当父母"

2. **角色会流动**
   - 同一句话可能让父亲从迫害者变成受害者
   - 设计 Storylet 时要考虑"角色转换"的触发点

3. **打破三角的路径**
   - 不是"解决问题"，而是"改变互动模式"
   - 玩家可以说："我不想当你们之间的传话筒了"

## 三、推荐故事线结构

综合以上参考，FacadeRemake 采用以下结构：

### Landmark 设计（4个主线锚点 + 4个结局，已在代码中实现）

> 代码文件：`data/default_landmarks.py`

```
Landmark 0: lm_0_surface — 表面平静
  场景：儿子刚到家，晚餐刚开始，一切看起来正常
  叙事目标：建立"曾经正常"的家庭氛围，埋下暗流
  推进条件：phone_incident=True 或 8 轮后强制推进
  信息防火墙：forbidden_reveals = ["money", "debt", "loan"]

Landmark 1: lm_1_cracks — 裂缝
  触发条件：phone_incident=True
  叙事目标：让玩家确认"确实有问题"，察觉父亲异常
  推进条件：mother_slipped=True 或 15 轮后强制推进
  信息防火墙：forbidden_reveals = ["thirty_thousand", "debt_amount"]

Landmark 2: lm_2_revelation — 真相
  触发条件：mother_slipped=True
  叙事目标：秘密被揭露，三个人被迫面对债务问题
  推进条件：mother_revealed_timing=True 或 20 轮后强制推进
  信息防火墙：无（此阶段可以全部揭露）

Landmark 3: lm_3_confrontation — 摊牌
  触发条件：mother_revealed_timing=True
  叙事目标：三个人把话说开，导向不同结局
  推进条件：mother_decided=True 或 30 轮后强制推进
```

### 结局设计（4个，已在代码中实现）

```
结局 A — 诚实的开始 (ending_honest_beginning)
  条件：father_apologized=True AND mother_decided=True
  特点：父亲道歉，母亲给了时间。不是和解，但至少不再撒谎了。

结局 B — 无解 (ending_unresolved)
  条件：marriage_tension >= 4
  特点：问题说出来了，但没有人解决。各自退回各自角落。

结局 C — 儿子的角色 (ending_son_mediated)
  条件：player_mediated=True AND mother_decided=True
  特点：儿子起到桥梁作用，父母意识到孩子长大了。

结局 D — 沉默 (ending_default)
  条件：兜底结局（其他条件均不满足时）
  特点：最压抑的结局，什么都没改变，但什么都暴露了。
```

## 四、World State 核心变量设计

> 以下变量已在 `core/world_state.py` 和 `main.py._init_world_state()` 中实现。

```python
# 数值型状态（Qualities）
{
    "marriage_tension": 0.0,       # 婚姻紧张度 0-10，控制叙事节奏
    "conversation_turns": 0,        # 对话总轮数
    "father_deflection_count": 0,   # 父亲回避次数（影响 Salience 评分）
    "story_phase": 0,               # 叙事阶段 0/1/2/3（与 Landmark 同步）
}

# 叙事进度标记（Flags）
{
    # Act1：到达与晚餐
    "arrived": False,               # 儿子已到家（由 sl_welcome_home 触发）
    "dinner_started": False,        # 晚餐开始（由 sl_dinner_table_silence 触发）
    "phone_incident": False,        # 电话事件（触发 lm_0→lm_1 推进）
    "mother_extra_care": False,     # 母亲过度关心
    "father_work_excuse": False,    # 父亲用工作搪塞

    # Act2：真相揭露
    "player_asked_wrong": False,    # 玩家开始追问
    "mother_slipped": False,        # 母亲说漏嘴（触发 lm_1→lm_2 推进）
    "secret_exposed": False,        # 秘密完整曝光
    "father_explained": False,      # 父亲作出解释
    "mother_revealed_timing": False,# 母亲揭示她早就知道（触发 lm_2→lm_3）
    "tuition_mentioned": False,     # 学费问题被提及

    # Act3：摊牌
    "confrontation_silence": False, # 摊牌后的沉默
    "father_apologized": False,     # 父亲道歉
    "mother_decided": False,        # 母亲做出决定（触发结局判断）
    "player_mediated": False,       # 玩家主动调解（影响结局）
}
```

**注意**：`NARRATIVE_REFERENCES.md` 中原有的抽象变量设计（如 `father_role`, `coping_strategy`）目前未在代码中实现，作为未来扩展方向保留参考。

## 五、关键设计原则

### 1. 没有真正的反派
参考《婚姻故事》：父母各自有合理的立场，玩家能理解双方。

### 2. 秘密是引擎，但不是终点
参考《玩偶之家》：揭露秘密不是为了"解决问题"，而是为了"改变认知"。

### 3. 玩家的声音很重要
参考《鱿鱼和鲸》：孩子不是被动的观察者，他们的反应会改变父母的互动。

### 4. 结局不是"修复"，而是"面对"
参考《长夜漫漫路迢迢》：有些问题没有解决方案，但"被看见"本身就是一种治愈。

### 5. 时间压缩增强张力
参考奥尼尔：把冲突压缩在"一顿晚餐"或"一个晚上"，避免叙事分散。
